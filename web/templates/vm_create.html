VM wizard
Опция нужна, если предыдущая попытка создания VM оборвалась и оставила хвост в /var/lib/virtuality/images.
Для обычных ISO выбирай x86_64 на ПК/VPS. Для Raspberry/Orange Pi образов обычно нужен ARM64/aarch64 и источник “Готовый диск”. Опция замены диска удаляет только /var/lib/virtuality/images/ИМЯ_VM.qcow2, если VM с таким именем ещё не существует.

Архитектура и источник

Открыть диски
x86_64 / amd64 Обычные ПК, серверы, большинство VPS и стандартные Ubuntu/Debian ISO.
ARM64 / aarch64 На x86-хосте работает как QEMU-эмуляция из готового диска .img/.qcow2. Это медленно, но честно.
Готовый диск Virtuality импортирует .img/.raw/.qcow2 в отдельный qcow2-диск VM, не изменяя оригинал. 
ISO x86: virt-install --arch x86_64 --cdrom ubuntu.iso
IMG ARM: qemu-img convert -O qcow2 image.img VM.qcow2
         virsh define /tmp/virtuality-VM-arm64.xml && virsh start VM
