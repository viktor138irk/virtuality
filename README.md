# Virtuality

**Virtuality** — лёгкая платформа виртуализации в стиле Proxmox на базе **KVM**, **QEMU**, **libvirt** и **Cockpit**.

Цель проекта — создать понятную, расширяемую и аккуратную систему для управления виртуальными машинами, хранилищами, сетевыми мостами, резервными копиями и будущей кластерной инфраструктурой.

> Текущий статус: ранняя стадия разработки.  
> Virtuality пока не является полноценной заменой Proxmox — это фундамент для создания собственной платформы виртуализации.

---

## Возможности

### Текущая база

- установка KVM/QEMU;
- настройка libvirt;
- веб-интерфейс Cockpit;
- модуль Cockpit Machines для управления виртуальными машинами;
- структура хранения ISO-образов, дисков виртуальных машин и бэкапов;
- libvirt storage pools;
- базовая подготовка firewall;
- чистая структура проекта для дальнейшей разработки.

### Планируемые возможности

- собственная веб-панель управления;
- управление жизненным циклом виртуальных машин;
- загрузка и управление ISO-образами;
- обзор хранилищ;
- планировщик резервного копирования;
- менеджер сетевых bridge-интерфейсов;
- роли пользователей и права доступа;
- диагностика ноды;
- система установки и автообновления;
- архитектура с заделом под кластер.

## Поддерживаемые системы

Целевые системы:

- Debian 12;
- Ubuntu Server 22.04;
- Ubuntu Server 24.04.

Рекомендуемая конфигурация сервера:

- чистая установка сервера;
- включённая аппаратная виртуализация в BIOS/UEFI;
- от 2 ядер CPU;
- от 4 GB RAM;
- SSD или NVMe-хранилище.
