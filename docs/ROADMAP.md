# Roadmap Virtuality

Документ фиксирует направление разработки, чтобы проект не превратился в хаотичный комбайн.

## v0.1 — Фундамент ноды

Цель: получить рабочую базу для запуска виртуализации.

Задачи:

- базовый установщик `install_virtuality_node.sh`;
- установка KVM/QEMU/libvirt;
- установка Cockpit и Cockpit Machines;
- создание директорий хранения;
- создание libvirt storage pools;
- базовая настройка firewall;
- подготовка GitHub-синхронизации;
- документация проекта.

## v0.2 — Сеть и диагностика

Цель: безопасно подготовить сетевую основу.

Задачи:

- диагностический скрипт ноды;
- определение активного сетевого интерфейса;
- безопасная настройка bridge `br0`;
- резервная копия сетевых конфигов перед изменением;
- откат сетевой конфигурации;
- улучшение логов установщика.

## v0.3 — Прототип панели

Цель: начать собственную панель управления.

Задачи:

- лёгкая веб-панель;
- авторизация;
- список виртуальных машин;
- статус VM;
- старт/стоп/перезагрузка VM через libvirt;
- отображение storage pools;
- базовый журнал событий.

## v0.4 — Создание VM

Цель: добавить мастер создания виртуальных машин.

Задачи:

- менеджер ISO;
- форма создания VM;
- выбор CPU/RAM/disk;
- выбор ISO;
- выбор сети;
- создание qcow2-диска;
- запуск установки ОС;
- базовая консоль.

## v0.5 — Обновления и бэкапы

Цель: сделать платформу обслуживаемой.

Задачи:

- автообновлятор;
- резервная копия конфигов перед обновлением;
- история обновлений;
- ручной backup VM;
- список бэкапов;
- восстановление из backup.

## v0.6 — Пользователи и роли

Цель: подготовить систему к реальному использованию несколькими пользователями.

Задачи:

- роли: admin, operator, viewer;
- ограничения действий по ролям;
- журнал действий пользователей;
- базовые настройки безопасности.

## v0.7 — Мониторинг

Цель: видеть состояние ноды без терминального шаманства.

Задачи:

- CPU/RAM/disk usage;
- статус libvirt;
- статус Cockpit;
- количество VM;
- состояние storage pools;
- алерты;
- Telegram/email-уведомления в будущем.

## v1.0 — Первый стабильный релиз

Цель: рабочая одиночная нода Virtuality.

Минимальный состав:

- установка одной командой;
- управление VM;
- создание VM;
- сеть bridge;
- ISO manager;
- backup/restore;
- web dashboard;
- базовая безопасность;
- документация;
- обновлятор.
